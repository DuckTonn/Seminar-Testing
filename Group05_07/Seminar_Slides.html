Tính năng nổi bật của Postman
Postman là công cụ phổ biến nhất hiện nay cho phát triển & kiểm thử API — giao diện trực quan cho cả Developer và QA.
Áp dụng cho EShop
1
Import OpenAPI/Swagger
Chuyển spec thành collection ngay lập tức
2
Environment & Auth Variables
Quản lý biến môi trường linh hoạt
3
Test Scripts (JavaScript)
Viết kịch bản kiểm thử ngay trong app
4
Collection Runner + Newman
Chạy hàng loạt, tích hợp CI/CD
Vì sao QA & Dev cùng dùng được
Giao diện trực quan (UI)
Dễ tiếp cận cho cả người mới lẫn Developer có kinh nghiệm, không cần viết code để bắt đầu.
Có thể lập trình được
Test Script bằng JavaScript giúp kiểm tra logic nghiệp vụ sâu hơn ngoài status code.
Sẵn sàng CI/CD
Newman cho phép chạy Collection tự động trong pipeline, không cần mở UI thủ công.

Độ bao phủ Test Cases
✓ Happy Path
Đảm bảo luồng xử lý thành công đúng nghiệp vụ (Status 200, schema chuẩn).
✗ Negative Path
Bắt lỗi các trường hợp token hết hạn, sai mật khẩu, tài khoản bị khóa.
⚠ Edge Cases
Xử lý các tình huống dị biệt: body rỗng, quantity âm, ID không hợp lệ.

Postman Failure Modes — điểm mù của tool
Các lỗi thường gặp do công cụ, dễ gây hiểu lầm cho người test.
Failure mode Mô tả
Sai kiểu dữ liệu API trả về String, nhưng logic test yêu cầu Number. Báo lỗi mơ hồ khiến QA lầm tưởng API sai nghiệp vụ.
Quên chọn Environment Gửi URL hoặc Token chứa chuỗi chưa được parse (VD: {{baseUrl}}), dẫn đến API sập nhưng mất thời gian debug sai hướng.
Silent Failure trong Script Viết sai cú pháp JavaScript, Postman bỏ qua toàn bộ script bên dưới nhưng UI không báo đỏ — vẫn hiện "Pass" (ảo giác Pass).
Bài học: Luôn kiểm tra kỹ biến môi trường và sử dụng linter / console.log trong Postman để debug script.
